# WordFlow 语音识别错误分析日志

## 错误信息

```
识别失败: Value cannot be null. (Parameter 'source')
```

---

## 问题原理说明

### 什么是 "Value cannot be null"？

这是 C# 中的 **空引用异常**（NullReferenceException）。意思是程序尝试使用一个为 `null`（空）的对象。

### 在这个语音识别流程中，可能发生在哪里？

```
┌─────────────────────────────────────────────────────────────┐
│                     语音识别流程                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 录音    →   2. 提取特征   →   3. ONNX推理   →   4. 解码  │
│     ✅            ✅              ❓ 可能出错      ❓ 可能出错  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 可能的出错点

| 位置 | 原因 | 说明 |
|------|------|------|
| **词汇表加载** | `tokens.txt` 格式不对 | 文件为空或格式错误，导致 `_tokens` 为 null |
| **归一化参数** | `am.mvn` 解析失败 | 均值/方差数组为 null |
| **ONNX 推理** | 模型输出为空 | 输入维度仍有问题，返回 null |
| **解码阶段** | 输出张量为空 | `results.First()` 可能抛出异常 |

## 当前代码的问题分析

### 问题1: 词汇表解析可能不正确

`tokens.txt` 的实际格式可能是：
```
<blank>
<sos/eos>
的
是
了
我
...
```

而代码中没有处理特殊标记。

### 问题2: 特征提取过于简化

当前只是简单复制音频数据，不是真正的 **FBank 特征**。Paraformer 需要的是经过复杂计算的特征向量。

### 问题3: 错误处理不够详细

出错时不知道具体是哪一步为空。

### 方案A: 添加详细日志（先做这个）

### 方案B: 使用正确的特征提取

## 推荐的调试步骤

### 步骤1: 检查模型文件

确认以下文件存在且不为空：
```
H:\WordFlow\Models\paraformer-onnx\
├── model_quant.onnx    (应该 ~100MB)
├── tokens.txt          (应该有几千行)
├── am.mvn              (应该有 Mean 和 Vars)
└── asr.yaml            (配置文件)
```

### 步骤2: 检查 tokens.txt 内容

用记事本打开 `tokens.txt`，看看：
- 第一行是不是 `<blank>`
- 第二行是不是 `<sos/eos>`
- 后面是不是中文汉字

### 步骤3: 检查 am.mvn 内容

应该有类似：
```
<LearnRateCoef> 1 </LearnRateCoef>
<Mean> 1.2 3.4 5.6 ... </Mean>
<Vars> 0.1 0.2 0.3 ... </Vars>
```

## 核心问题总结

> **当前代码的特征提取是"伪代码"，只是简单复制音频数据，不是真正的 FBank 特征提取。**

Paraformer 模型期望的输入是：
- 经过预加重、分帧、加窗的音频
- 经过 FFT 变换到频域
- 经过 Mel 滤波器组
- 取对数得到的 80 维 FBank 特征
- 再拼接成 560 维（可能包含上下文）

简化版无法工作，需要：
1. 接入完整的特征提取库，或
2. 调用 Python 后端进行推理，或
3. 参考现有的 C++ 实现
