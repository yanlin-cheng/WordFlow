### 问题1: 输入维度不匹配

**时间**: 2025-02-08 下午  
**错误信息**:
```
识别失败: [ErrorCode:InvalidArgument] Got invalid dimensions for input: speech 
index: 2 Got: 80 Expected: 560
```

**原因分析**:  
Paraformer 模型期望输入特征维度是 **560**，但代码中使用的是 **80**（标准 FBank 维度）。

**解决方案**:  
将特征维度从 80 改为 560：
```csharp
int numFeatures = 560;  // 原为 80
```

**状态**: ✅ 已修复

### 问题3: 模型输出类型错误（已解决）

**时间**: 2025-02-08 晚间  

**错误现象**:
```
[Recognize] 输出Value类型: DenseTensor`1
[Recognize] 成功转换为 Tensor<long>   ← 实际上这一步返回null
[Recognize] 错误: 输出张量为null!
```

**根本原因**:  
Paraformer 模型输出的是 **logits**（浮点概率分布），不是直接的 token IDs。

```
模型输出: logits [batch, time, vocab_size]  (float)
          ↓
需要:     argmax 在最后一维取最大值索引
          ↓
得到:     token IDs [batch, time]  (long)
```

**解决方案**:  
添加 `ArgMax` 方法，在 logits 上取最大值索引：

```csharp
// 读取float类型的logits
var logits = firstResult.AsTensor<float>();

// 取argmax得到token IDs
var tokenIds = ArgMax(logits);

// 解码
return Decode(tokenIds);
```

**关键代码**:
```csharp
private long[] ArgMax(Tensor<float> logits)
{
    // 对每一帧，在vocab维度取最大值的索引
    for (int t = 0; t < timeSteps; t++)
    {
        float maxVal = float.MinValue;
        int maxIdx = 0;
        
        for (int v = 0; v < vocabSize; v++)
        {
            if (logits[0, t, v] > maxVal)
            {
                maxVal = logits[0, t, v];
                maxIdx = v;
            }
        }
        result[t] = maxIdx;
    }
    return result;
}
```

**状态**: ✅ 已修复

### 问题4: 调试信息未输出到VS输出窗口

**时间**: 2025-02-08 晚间  

**现象**:  
使用 `System.Diagnostics.Debug.WriteLine()` 时，调试信息在 VS 的输出窗口中不显示或显示不完整。

**根本原因**:  
`Debug.WriteLine()` 只在调试模式下工作，且可能被 VS 的输出窗口过滤。

**解决方案**:  
创建文件日志记录器 `Logger.cs`，将所有调试信息写入文件：

```csharp
public static class Logger
{
    private static readonly string LogFilePath = Path.Combine(
        AppDomain.CurrentDomain.BaseDirectory, 
        "debug_log.txt");
    
    public static void Log(string message)
    {
        var timestamp = DateTime.Now.ToString("yyyy-MM-dd HH:mm:ss.fff");
        var logLine = $"[{timestamp}] {message}{Environment.NewLine}";
        File.AppendAllText(LogFilePath, logLine);
    }
}
```

**使用方式**:  
```csharp
Utils.Logger.Log("[Recognize] 开始识别...");
```

**日志位置**:  
`H:\WordFlow\bin\Debug\net10.0-windows\debug_log.txt`

**状态**: ✅ 已修复

### 问题5: 识别结果错误（乱码/重复）

**时间**: 2025-02-08 晚间  

**现象**:  
识别结果全是乱码和重复的"是"：
```
被有三是是是是是是是是是是是是是是是有有有有的诉有
```

**根本原因**:  
C# 代码中的 **FBank特征提取** 是简化版，只是简单复制音频数据，没有真正的信号处理。

**真正的FBank需要**:
- 预加重（高通滤波）
- 分帧（25ms帧，10ms移）
- 加窗（汉明窗）
- FFT（快速傅里叶变换）
- Mel滤波器组
- 对数运算
- CMVN归一化

**解决方案**:  
创建 **Python ASR服务**，使用正确的特征提取算法。

**项目结构**:
```
PythonASR/
├── asr_server.py          # HTTP服务，完整FBank+ONNX
├── test_asr.py            # 测试脚本
├── requirements.txt       # Python依赖
└── start_server.bat       # 启动脚本
```

**启动方式**:
```powershell
cd H:\WordFlow\PythonASR
python asr_server.py
```

**架构**:
```
C# WPF  →  HTTP  →  Python ASR服务  →  ONNX推理  →  文字结果
```

**状态**: ✅ Python服务已创建，待测试
