## 三、具体实现方法

### 3.1 全局热键监听

**原理**：使用 Windows 底层键盘钩子 (WH_KEYBOARD_LL)

```csharp
// Services/GlobalHotkeyService.cs
public class GlobalHotkeyService : IDisposable
{
    private const int WH_KEYBOARD_LL = 13;
    private const int VK_CAPITAL = 0x14;  // CapsLock
    
    // 事件：录音键按下/释放
    public event EventHandler? RecordingKeyPressed;
    public event EventHandler? RecordingKeyReleased;
    
    // 设置钩子
    private IntPtr SetHook(LowLevelKeyboardProc proc)
    {
        using (var curProcess = Process.GetCurrentProcess())
        using (var curModule = curProcess.MainModule)
        {
            return SetWindowsHookEx(WH_KEYBOARD_LL, proc,
                GetModuleHandle(curModule?.ModuleName ?? ""), 0);
        }
    }
}
```

**关键点**：
- 钩子必须在 STA 线程设置
- 需要调用 `CallNextHookEx` 传递事件
- 程序退出时必须 `UnhookWindowsHookEx`

### 3.3 核心：文字上屏实现

**最终精简版代码**（仅 10 行核心逻辑）：

```csharp
// MainWindow.xaml.cs
private async Task SendTextToExternalWindowAsync(IntPtr targetWindow, string text)
{
    // 1. 激活目标窗口
    BringWindowToForeground(targetWindow);
    await Task.Delay(50);
```

### 3.4 窗口焦点管理

**为什么需要？**
- 录音时 WordFlow 不能抢夺焦点
- 输入时必须确保目标窗口在前台

```csharp
// 保存录音前的焦点窗口
private IntPtr _lastForegroundWindow;

private void StartRecording()
{
    // 在录音前保存当前焦点
    _lastForegroundWindow = GetForegroundWindow();
    var myHandle = new WindowInteropHelper(this).Handle;
    
    // 只有焦点在其他窗口时才保存
    if (_lastForegroundWindow != myHandle)
    {
        // 开始录音...
    }
}

// 恢复窗口到前台
private void BringWindowToForeground(IntPtr hWnd)
{
    uint currentThreadId = GetCurrentThreadId();
    uint windowThreadId = GetWindowThreadProcessId(hWnd, out _);
    
    // 如果线程不同，附加输入队列
    if (currentThreadId != windowThreadId)
    {
        AttachThreadInput(windowThreadId, currentThreadId, true);
    }
    
    // 激活窗口
    BringWindowToTop(hWnd);
    SetForegroundWindow(hWnd);
    ShowWindow(hWnd, SW_SHOW);
    
    // 分离输入队列
    if (currentThreadId != windowThreadId)
    {
        AttachThreadInput(windowThreadId, currentThreadId, false);
    }
}
```

**解决方案**：

### 4.2 焦点抢夺问题

**现象**：文字总是输入到 WordFlow 自己，而不是目标窗口

**原因**：更新 WordFlow 编辑器时，TextBox 自动获取焦点

**解决方案**：
```csharp
private void UpdateEditorWithoutFocus(string text)
{
    // 保存当前焦点
    var currentFocus = GetForegroundWindow();
    
    // 更新文本
    EditorTextBox.Text = EditorTextBox.Text.Insert(position, text);
    
    // 如果焦点被抢走，恢复回去
    if (GetForegroundWindow() != currentFocus)
    {
        BringWindowToForeground(currentFocus);
    }
}
```

---

### 4.3 输入法兼容性

**问题**：模拟按键时可能触发输入法，导致输入异常

**解决方案**：
- 使用剪贴板粘贴（绕过输入法）
- 不逐字模拟按键
- 一次性粘贴整段文字

---

### 4.4 延迟调优

| 操作 | 延迟 | 说明 |
|------|------|------|
| 激活窗口 | 50ms | 确保窗口响应 |
| 粘贴后等待 | 50ms | 确保粘贴完成 |
| 字符间隔 | 20ms | 逐字输入时使用 |

**注意**：延迟太短可能导致输入失败，太长体验不佳。

### 4.5 剪贴板冲突
