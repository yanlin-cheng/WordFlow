## 问题 1：最小化后无法重新唤起

### 现象描述
- 窗口最小化到系统托盘后，按热键（CapsLock 或 右 Alt）无法唤起程序
- 程序完全无响应，无法恢复窗口

### 已尝试的修复方案

#### 尝试 1：使用 Application.Dispatcher.BeginInvoke
**修改位置**: `MainWindow.xaml.cs` - `OnRecordingKeyPressed/OnRecordingKeyReleased`
```csharp
// 修改前
Dispatcher.Invoke(() => { ... });

// 修改后
Application.Current.Dispatcher.BeginInvoke(() => { ... });
```
**结果**: ❌ 无效

#### 尝试 2：修改 StartRecording 中的 Dispatcher 调用
**修改位置**: `MainWindow.xaml.cs`
- `ShowRecordingNotification()` 改用 `Application.Current.Dispatcher.BeginInvoke()`
- `StartRecordingTimer()` 使用 `Application.Current.Dispatcher` 创建定时器
- 错误提示也改用 `BeginInvoke`

**结果**: ❌ 仍然无效

### 可能原因分析
1. **窗口隐藏后 Dispatcher 停止工作** - WPF 窗口隐藏后，其 Dispatcher 消息循环可能被挂起
2. **低级键盘钩子回调阻塞** - 钩子回调中的任何同步操作都可能导致问题
3. **托盘服务与主窗口耦合问题** - 窗口句柄隐藏后某些 Windows API 调用失败

### 待尝试方案
1. 将热键处理完全移到 App.xaml.cs 层级，与窗口完全解耦
2. 使用 Windows 全局原子操作记录录音状态，避免依赖 Dispatcher
3. 考虑使用 Windows 服务或独立进程处理录音

## 问题 2：设置界面无法进入设置快捷键

### 现象描述
- 点击工具栏"⚙️ 设置"按钮后，设置窗口无法打开
- 程序可能卡住或无响应

### 相关代码
**按钮事件**: `MainWindow.xaml.cs` - `SettingsButton_Click`
```csharp
private void SettingsButton_Click(object sender, RoutedEventArgs e)
{
    var settingsService = (App.Current as App)?.Settings;
    var hotkeyService = _hotkeyService;
    
    if (settingsService != null && hotkeyService != null)
    {
        var settingsWindow = new Views.SettingsWindow(settingsService, hotkeyService)
        {
            Owner = this
        };
        settingsWindow.ShowDialog();
    }
}
```

### 可能原因
1. **App.Settings 为空** - `App.xaml.cs` 中的 `Settings` 属性可能未正确初始化
2. **_hotkeyService 为空** - 热键服务初始化失败或在某些场景下为空
3. **SettingsWindow 构造函数异常** - XAML 加载或依赖注入问题
4. **ShowDialog 阻塞** - 模态对话框在特定条件下阻塞主线程

### 需要检查的点
- [ ] `App.xaml.cs` 中 `_settingsService` 是否正确初始化
- [ ] `MainWindow` 中 `_hotkeyService` 是否在 `InitializeGlobalHotkey` 中正确赋值
- [ ] `SettingsWindow.xaml` 是否存在绑定错误
- [ ] 是否存在循环依赖或死锁
