# WordFlow 安装包打包说明文档

本文档详细说明 WordFlow 安装包的打包内容、原因和排除项。

## 一、项目结构概览

```
WordFlow_V1.0.0/
├── WordFlow.csproj          # .NET 项目文件
├── build_installer.bat      # 安装包构建脚本
├── Installer/
│   └── WordFlowSetup.iss    # Inno Setup 安装脚本
├── bin/Release/             # 编译输出目录
├── Output/Installer/        # 安装包输出目录
│   ├── WordFlow/            # 打包文件临时目录
│   └── WordFlow_Setup.exe   # 最终安装包
├── PythonASR/               # Python 语音识别服务
│   ├── python/              # 嵌入式 Python 环境
│   ├── models/              # 语音识别模型（不打包）
│   └── *.py                 # Python 脚本
└── Data/                    # 配置文件
```

## 二、需要打包的文件

### 2.1 .NET 主程序文件

| 文件类型 | 来源目录 | 打包原因 |
|---------|---------|---------|
| `WordFlow.exe` | `bin/Release/net8.0-windows/` | 主程序可执行文件 |
| `*.dll` | `bin/Release/net8.0-windows/` | 程序集依赖库 |
| `*.runtimeconfig.json` | `bin/Release/net8.0-windows/` | .NET 运行时配置 |
| `runtimes/` | `bin/Release/net8.0-windows/runtimes/` | 原生库（SQLite、ONNX Runtime） |
| `win-x64/` | `bin/Release/net8.0-windows/win-x64/` | .NET 8 运行时（框架依赖） |

**打包原因**：这些是 WordFlow 应用程序运行所必需的核心文件。

### 2.2 PythonASR 脚本文件

| 文件 | 打包原因 |
|-----|---------|
| `asr_server.py` | 语音识别服务主程序 |
| `download_model.py` | 模型下载脚本 |
| `setup_model.py` | 模型配置脚本 |
| `start_server.bat` | 服务启动脚本 |
| `requirements.txt` | Python 依赖列表 |
| `*.md` | 使用说明文档 |

**打包原因**：这些脚本用于启动语音识别服务和下载模型。

### 2.3 嵌入式 Python 环境（完整版）

| 文件/目录 | 打包原因 |
|----------|---------|
| `python.exe` | Python 解释器 |
| `python3.dll`, `python311.dll` | Python 核心 DLL |
| `libcrypto-3.dll`, `libssl-3.dll` | OpenSSL 库 |
| `sqlite3.dll` | SQLite 支持 |
| `vcruntime140.dll`, `vcruntime140_1.dll` | VC++ 运行时 |
| `python311._pth` | Python 路径配置 |
| `Lib/site-packages/` | 完整的 Python 包（包含 sherpa_onnx、numpy 等） |
| `Scripts/*.dll` | Python 扩展库 DLL |

**打包原因**：用于运行 Python 语音识别脚本，包含所有运行时依赖。

### 2.4 配置文件

| 文件 | 打包原因 |
|-----|---------|
| `Data/models.json` | 模型配置信息 |

## 三、不打包的文件及原因

### 3.1 模型文件（`PythonASR/models/`）

**不打包原因**：
- 体积过大（约 200MB）
- 可通过网络下载
- 用户在程序内根据需要下载

**处理方式**：
- 安装包中保留空的 `models/` 目录
- 用户在程序设置中下载模型
- 下载后模型存储于 `{app}\PythonASR\models\`

### 3.2 Python 虚拟环境（`PythonASR/venv/`）

**不打包原因**：
- 包含大量开发相关文件
- 体积大（约 500MB+）
- 包含不必要的调试符号和测试文件

**处理方式**：
- 使用精简的嵌入式 Python（`PythonASR/python/`）
- 仅复制运行时必需的库

### 3.3 Python 缓存文件（`__pycache__/`）

**不打包原因**：
- 可在首次运行时自动生成
- 增加安装包体积

### 3.4 编译中间文件（`obj/`）

**不打包原因**：
- Visual Studio 编译生成的临时文件
- 不包含运行时必需内容

### 3.5 安装包输出（`Output/Installer/WordFlow/`）

**不打包原因**：
- 这是构建脚本生成的临时目录
- 由 Inno Setup 直接读取

### 3.6 发布测试目录（`publish_*/`）

**不打包原因**：
- 测试用的发布版本
- 与安装包构建无关

### 3.7 上传文件（`上传文件/`）

**不打包原因**：
- 用于上传到应用商店的预编译版本
- 与安装包构建无关

### 3.8 WordFlowInstaller 目录

**不打包原因**：
- 独立的安装器项目
- 与 Inno Setup 安装包无关

## 五、构建流程

### 5.1 执行 `build_installer.bat`

```
1. 清理旧的 Output/Installer/WordFlow 目录
2. 复制 .NET 编译输出到 Output 目录
3. 复制 PythonASR 脚本（排除 venv、models）
4. 复制完整的 Python 环境
5. 复制 Data 配置文件
6. 调用 Inno Setup 编译安装包
```

### 5.2 Inno Setup 处理

`Installer/WordFlowSetup.iss` 从 `Output/Installer/WordFlow/` 读取文件，打包成单个 `WordFlow_Setup.exe`。

### 5.3 安装包内容

最终安装包包含：
- .NET 主程序及依赖
- 完整 Python 环境（约 60MB）
- Python 脚本和配置文件
- 空的 models 目录

**预计安装包大小**：约 80-100MB（压缩后）

## 六、安装流程说明

### 6.1 .NET Runtime 检测与安装

安装程序会在点击"安装"前检测系统是否已安装 .NET 8 Runtime：

| 状态 | 处理方式 |
|-----|---------|
| 已安装 | 直接继续安装 |
| 未安装 | 弹出提示，询问是否自动下载安装（约 60MB） |

**用户选择"是"**：
- 下载 .NET 8 Runtime（约 60MB）
- 静默安装（无用户交互）
- 安装完成后继续 WordFlow 安装

**用户选择"否"**：
- 记录日志
- 显示警告提示，提供 .NET 下载链接
- 继续安装（但程序安装后无法运行）

### 6.2 语音识别模型下载

安装完成后：
- 显示提示，告知用户在程序内下载模型
- 首次运行时自动启动 Python 语音识别服务
- 用户在程序设置中选择并下载模型（约 200MB）
- 模型存储于 `{app}\PythonASR\models\`

### 6.3 Python 服务自动启动

程序启动时：
1. 检测 Python 服务是否运行
2. 如未运行，自动执行 `start_server.bat`
3. 等待服务启动完成后连接

## 七、常见问题及解决

### 7.1 SQLite 连接错误

**错误信息**：`The type initializer for 'Microsoft.Data.Sqlite.SqliteConnection' threw an exception`

**原因**：SQLite 原生库未正确加载

**解决方法**：
1. 确保 `e_sqlite3.dll` 存在于安装目录
2. 检查 `runtimes/win-x64/native/` 目录完整性
3. 重新安装 .NET 8 Runtime

### 7.2 Python 服务连接失败

**错误信息**：`由于目标计算机积极拒绝，无法连接 (127.0.0.1:5000)`

**原因**：Python 服务未启动

**解决方法**：
1. 手动运行 `{app}\PythonASR\start_server.bat`
2. 检查 Python 环境是否完整
3. 查看命令行窗口中的错误信息

### 7.3 模型下载失败

**错误代码 2**：通常是网络问题或路径问题

**解决方法**：
1. 检查网络连接
2. 确保 models 目录有写入权限
3. 手动下载模型文件并解压到 `{app}\PythonASR\models\`

## 八、手动编译安装包

如果自动脚本失败，可手动编译：

1. 确保已安装 [Inno Setup 6](https://jrsoftware.org/isdl.php)
2. 打开 `Installer\WordFlowSetup.iss`
3. 在 Inno Setup 中点击 "Compile"
4. 生成的安装包位于 `Output\Installer\WordFlow_Setup.exe`
