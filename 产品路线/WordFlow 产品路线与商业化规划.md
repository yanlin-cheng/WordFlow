## 一、产品愿景与定位

### 1.1 产品愿景

**让语音输入更智能、更便捷**

WordFlow 致力于成为专业用户首选的语音输入工具，通过本地 AI 识别与云端智能增强的结合，提供高精度、低延迟、隐私安全的语音转文字体验。

### 1.2 产品定位

| 维度 | 定位 |
|------|------|
| **目标用户** | 需要大量文字输入的专业人士和内容创作者 |
| **核心价值** | 离线优先、本地识别、云端增强、词库生态 |
| **差异化** | 与竞品相比：更轻量、更隐私、更具性价比 |
| **价格定位** | 基础功能免费，增值服务低价普惠 |

### 1.3 目标用户画像

#### 核心用户群体

| 用户类型 | 特征 | 需求 | 付费意愿 |
|---------|------|------|---------|
| **医生** | 需要快速记录病历、处方 | 医学术语识别、隐私保护 | 高 |
| **律师** | 需要记录案情、法律文书 | 法律术语识别、准确性 | 高 |
| **记者/作家** | 需要快速记录灵感、采访 | 长文本输入、词库同步 | 中 |
| **学生/研究者** | 需要记录笔记、论文 | 学术术语、多语言支持 | 中 |
| **UP 主/博主** | 需要创作脚本、字幕 | 快速输入、词库分享 | 中 |
| **普通上班族** | 日常办公、聊天输入 | 简单易用、免费基础功能 | 低 |

### 1.4 竞品分析

| 竞品 | 优势 | 劣势 | WordFlow 对策 |
|------|------|------|--------------|
| **Typeless** | 品牌知名、AI 智能 | 臃肿 (473MB)、依赖云端 | 轻量 (100MB 内)、离线优先 |
| **CapsWriter-Offline** | 开源免费、简单 | 无商业功能、UI 简陋 | 专业界面、词库生态、云服务 |
| **讯飞输入法** | 识别率高、品牌强 | 需要联网、隐私担忧 | 本地识别、隐私保护 |
| **微信语音转文字** | 方便、免费 | 准确率一般、功能单一 | 专业词库、个性化学习 |

### 2.2 词库交易模式

#### 定价策略

| 操作 | 价格 | 说明 |
|------|------|------|
| **上传词库** | ¥1-2 元/次 | 创作者设置价格，平台抽成 20% |
| **下载词库** | ¥1-2 元/次 | 按次付费，永久使用 |
| **收益分成** | 创作者 80% | 平台抽成 20%，用于服务器和运营成本 |

#### 分成示例

```
词库定价：¥2.00

下载 100 次：
├── 总收入：¥200
├── 创作者分成：¥160 (80%)
└── 平台收入：¥40 (20%)
```

### 3.2 安装包策略

**单一安装包，运行时权限控制**

```
┌─────────────────────────────────────────────────────┐
│              WordFlow_Setup.exe                     │
│              (统一安装包，约 60-100MB)               │
│                                                     │

│  ├── 基础功能模块 (所有人可用)                      │
```

### 3.3 为什么只需要一个安装包？

| 原因 | 说明 |
|------|------|
| **维护简单** | 不需要维护两套安装逻辑和更新通道 |
| **代码统一** | 版本号、更新逻辑一致，减少 bug |
| **用户体验** | 用户不需要选择"下载哪个版本" |
| **转化方便** | 基础用户→付费用户，无需重新安装 |
| **更新可控** | 更新服务器由你控制，确保推送可达 |

### 4.2 开源协议选择

#### 开源部分：MIT License

```
MIT License 优势：
- 最大化社区参与度
- 容易获得用户信任
- 有利于传播和采用
- 允许商业使用（但闭源部分仍受保护）
```

#### 闭源部分：专有许可

```
专有许可条款：
- 仅限个人使用
- 禁止反向工程
- 禁止用于训练或其他 AI 用途
- 保留所有权利
```

### 4.3 代码结构划分

```
WordFlow/
├── 🟢 开源模块
│   ├── WordFlow.Core/           # 核心功能库
│   ├── WordFlow.Services.Local/ # 本地服务
│   ├── WordFlow.UI/             # 用户界面
│   └── PythonASR/               # Python ASR 服务
│
├── 🔴 闭源模块
│   ├── WordFlow.Services.Cloud/ # 云服务 (独立 DLL)
│   ├── WordFlow.Services.Auth/  # 认证服务 (独立 DLL)
│   └── WordFlow.Services.Market/# 词库市场 (独立 DLL)
│
└── 编译配置
    ├── 开源版本 → 不包含闭源 DLL
    └── 发布版本 → 包含所有 DLL
```

### 4.4 开源 vs 发布版关系

```
开源版 = GitHub 源码 (用户可自行编译)
    ↓
包含开源模块源码
不包含闭源模块源码 (闭源模块以预编译 DLL 形式提供)

发布版 = 安装包 (普通用户下载使用)
    ↓
包含所有功能模块
用户登录后根据权限解锁功能
```

### 4.5 为什么不需要加壳/反破解？

| 原因 | 说明 |
|------|------|
| **核心价值在云端** | 词库交易、用户数据都在服务器 |
| **本地破解无意义** | 破解了也拿不到云词库，无法使用付费功能 |
| **开源部分不怕看** | MIT 协议允许随便看、随便用 |
| **闭源部分依赖服务器验证** | 没有账号，闭源 DLL 也白搭 |
| **节省成本** | 加壳/混淆需要额外成本和精力 |

### 5.2 用户系统架构

#### 数据库设计

```sql
-- 用户表
CREATE TABLE users (
    id UUID PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    last_login_at TIMESTAMP,
    subscription_type VARCHAR(20) DEFAULT 'free',
    subscription_expires_at TIMESTAMP
);

-- 订阅表
CREATE TABLE subscriptions (
    id UUID PRIMARY KEY,
    user_id UUID REFERENCES users(id),
    type VARCHAR(20) NOT NULL, -- 'professional' | 'enterprise'
    status VARCHAR(20) NOT NULL, -- 'active' | 'expired' | 'cancelled'
    started_at TIMESTAMP NOT NULL,
    expires_at TIMESTAMP NOT NULL,
    auto_renew BOOLEAN DEFAULT true
);

-- 词库表
CREATE TABLE vocabularies (
    id UUID PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    description TEXT,
    category VARCHAR(50),
    author_id UUID REFERENCES users(id),
    author_name VARCHAR(50),
    price DECIMAL(10,2) DEFAULT 0,
    is_public BOOLEAN DEFAULT false,
    is_premium BOOLEAN DEFAULT false,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    download_count INTEGER DEFAULT 0,
    rating DECIMAL(3,2) DEFAULT 0
);

-- 词库条目表
CREATE TABLE vocabulary_entries (
    id UUID PRIMARY KEY,
    vocabulary_id UUID REFERENCES vocabularies(id),
    word VARCHAR(100) NOT NULL,
    pinyin VARCHAR(100),
    definition TEXT,
    weight DECIMAL(5,4) DEFAULT 1.0
);

-- 下载记录表
CREATE TABLE download_records (
    id UUID PRIMARY KEY,
    user_id UUID REFERENCES users(id),
    vocabulary_id UUID REFERENCES vocabularies(id),
    downloaded_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    amount DECIMAL(10,2) NOT NULL
);

-- 收益分成表
CREATE TABLE revenue_shares (
    id UUID PRIMARY KEY,
    author_id UUID REFERENCES users(id),
    vocabulary_id UUID REFERENCES vocabularies(id),
    download_record_id UUID REFERENCES download_records(id),
    amount DECIMAL(10,2) NOT NULL, -- 创作者分成 (80%)
    platform_fee DECIMAL(10,2) NOT NULL, -- 平台抽成 (20%)
    status VARCHAR(20) DEFAULT 'pending', -- 'pending' | 'paid'
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- API 使用记录表
CREATE TABLE api_usage_records (
    id UUID PRIMARY KEY,
    user_id UUID REFERENCES users(id),
    api_type VARCHAR(50) NOT NULL,
    request_count INTEGER DEFAULT 0,
    cost DECIMAL(10,2) DEFAULT 0,
    recorded_date DATE NOT NULL
);

-- 订单表
CREATE TABLE orders (
    id UUID PRIMARY KEY,
    user_id UUID REFERENCES users(id),
    type VARCHAR(50) NOT NULL, -- 'subscription' | 'vocabulary' | 'api_package'
    amount DECIMAL(10,2) NOT NULL,
    status VARCHAR(20) NOT NULL, -- 'pending' | 'paid' | 'cancelled' | 'refunded'
    payment_method VARCHAR(50),
    paid_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```
